## 2. Agentic Pipeline Improvements

- [x] **Auto-curation in the weekly workflow** — Currently the workflow only scrapes candidates; it doesn't call `curator_agent.py` or `review_agent.py`. Wire them into the CI pipeline so candidates arrive pre-scored and pre-noted, requiring only a final human approve/reject.
- [ ] **Deduplication agent** — Before writing a candidate to `registry/candidates/`, check if it already exists in the registry by `id`, `url`, or name fuzzy-match. (Basic ID dedup already exists in `fetch_new.py`; extend to URL + name matching.)
- [x] **`update_entry.py` agent** — Periodically re-checks existing entries for stale `stars_approx`, deprecated URLs, or changed maturity. Runs monthly and opens a PR with diffs.
- [x] **Changelog / audit log** — Track when entries were promoted, who reviewed them, and what score they got. Stored in `registry/audit.json`.
